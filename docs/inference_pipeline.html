The dpbusd GEMM Kernel

Spatial Packing (8x speedup)

Pack 8 spatial positions into one YMM register instead of 8 partial sums. Each lane accumulates independently — no horizontal reduction needed.

Naive: hadd required

ymm = [s0 s1 s2 ... s7]
8 partial sums → reduce to 1

vs
Spatial: lane-independent

ymm = [p0 p1 p2 ... p7]
8 outputs, each lane done

14-OC Register Tiling

Inspired by disassembling ONNX Runtime's MLAS kernel. Keep 14 OC accumulators live in YMM registers: r0–r13 + 1 activation + 1 weight = 16 YMM total.

Inner loop: 14 × dpbusd = 448 MACs/iteration

Each dpbusd: 32 unsigned × 32 signed → 8 int32 accumulators

Specialized im2col (3×3 s1p1)

12 of 16 ternary convs use 3×3 stride-1 pad-1. Replaced per-element gather with per-row memcpy.

Before: 30% of runtime (scatter)
After: 7.4% of runtime (sequential)

Fused Epilogue Modes

Mode 0dequant → FP32
Mode 1dequant → ReLU → uint8
Mode 2dequant → +residual → ReLU → uint8

Zero intermediate buffers. int32 → final format in one pass.

Weight Representation

Storage: 2-bit I2_S

4 weights per byte, MSB first:

+1 → 0b01   0 → 0b00   -1 → 0b11

44.69 MB → 3.56 MB (92.2% reduction)

Runtime: INT8 + pack4

At init, 2-bit → int8 {-1, 0, +1}, pre-packed for dpbusd broadcast:

pack4(w0,w1,w2,w3) = w0 | w1<<8 | w2<<16 | w3<<24

Trades 4x storage at runtime for zero-cost access. Fits in L2.
